# gomoku

# 项目背景

五子棋（Gomoku）是一种经典的二人对弈棋类游戏，规则简单但策略深远。棋盘为15×15网格，黑白双方轮流落子，先在横、竖、斜任一方向形成连续五子的一方获胜。

本项目旨在实现一个具备较强棋力的五子棋AI，通过实践深入理解：

· 博弈树搜索与剪枝优化
· 启发式评估函数的设计
· 零和博弈中的决策理论

选择五子棋作为大作业主题，因为它完美契合课程中的搜索算法与数据结构知识，同时具有直观的视觉反馈，适合构建图形界面交互。

# 核心算法

1. Minimax搜索（极小极大搜索）

Minimax是博弈论中的核心算法，模拟双方在博弈树中轮流决策的过程：

· MAX层（AI回合）：选择评估值最大的走法（AI希望获胜）
· MIN层（对手回合）：选择评估值最小的走法（对手希望AI失败）

公式表达：

```
V(s) = max( V(child) )   if MAX turn
V(s) = min( V(child) )   if MIN turn
```

课程知识点对应： 树的深度优先遍历、递归回溯思想

---

2. Alpha-Beta剪枝

Alpha-Beta剪枝是对Minimax搜索的关键优化，通过维护两个参数：

参数 含义
α (Alpha) MAX层当前已知的最佳值（下界）
β (Beta) MIN层当前已知的最佳值（上界）

剪枝条件：

· 当 α ≥ β 时，当前分支不可能影响最终决策，直接剪除

优化效果： 在相同搜索深度下，可将搜索节点数减少约 70%~90%，使AI能在有限时间内探索更深层次。

课程知识点对应： 搜索剪枝策略、算法效率分析

---

3. 启发式评估函数

评估函数是AI"思考"的核心，通过量化棋盘局势给出评分。

特征检测模式
  
模式                        评分                                说明
五子连珠                 1,000,000                        必胜
活四                        100,000                           两端开放的四子，必败
冲四                        10,000                             一端开放的四子，必须防守
活三                        8,000                               两端开放的三子，巨大威胁
跳活三                    6,000                                X _ X _ X 模式
眠三                        1,000                               一端开放的三子

评估流程

```
对于每个棋子：
  1. 检查四个方向（横、竖、主对角、副对角）
  2. 统计连续棋子数量
  3. 检查两端是否开放（空位/边界/对方棋子）
  4. 根据"数量+开放端"组合计算评分
  5. AI得分 - 对手得分 × 1.2（防守倾向）
```

课程知识点对应： 多维数组遍历、特征提取与加权求和

---

4. 邻域候选搜索

在全棋盘搜索效率极低（15×15=225个位置），本项目采用邻域搜索优化：

```
候选位置 = 所有已有棋子周围2格范围内的空位
```

优化效果： 平均每步候选位置从225降至 20~40个，搜索效率提升 5~10倍。

课程知识点对应： 空间局部性原理、集合去重

---

5. 双威胁检测（关键战术）

AI能够识别并主动形成/防守组合攻击：

· 双活三：两个活三同时存在，对手无法同时防守 → 必胜
· 活三+冲四：一个活三和一个冲四，对手顾此失彼 → 必胜

检测逻辑：

```
对每个位置尝试落子后：
  统计所有方向的威胁（活三/冲四数量）
  如果 ≥ 2，判定为双威胁
```

课程知识点对应： 模式匹配、组合枚举

# 运行指南

鼠标点击 在棋盘交叉点落子
游戏模式 切换「人机对战」/「双人对战」
AI难度 1~5级，控制搜索深度（2~6层）
新游戏 重置棋盘
悔棋 人机模式撤回两步（AI+玩家），双人模式撤回一步

游戏规则

1. 黑棋先行（玩家执黑，AI执白）
2. 横/竖/斜任一方向形成五子连珠即获胜
3. 棋盘下满为平局
